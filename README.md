## StudyAI
### 🚀 AI 개인 프로젝트 스터디

개인 프로젝트 스터디 레포지토리입니다. 개별 폴더에서 각자의 프로젝트를 진행하며, PR을 통해 서로의 코드를 리뷰하고 성장합니다.

#### 📂 폴더 구조

이 레포지토리는 스터디원 각자의 독립된 작업 공간으로 나뉘어 있습니다. 모든 작업은 본인의 이름으로 생성된 폴더 내에서 진행해 주세요!

```text
.
├── 📁 Boyoung/       # 보영
├── 📁 Seohyun/       # 서현
├── 📁 Soonkyu/       # 순규
└── 📁 Yujin/         # 유진

```

### 🔄 스터디 진행 워크플로우
**우리 스터디는 `main` 브랜치에 직접 코드를 올리지 않고, 반드시 Pull Request (PR)와 코드 리뷰를 거쳐 병합(Merge)합니다.**

1. **브랜치 생성**: `main` 브랜치 최신 상태에서 본인의 작업 브랜치를 생성합니다.  
    브랜치명 규칙: `feature/본인이름-작업내용` (예: `feature/seohyun-ai-model`)  
2. **작업 및 커밋**: 본인의 개인 폴더 안에서 코드를 작성하고 커밋합니다.  
3. **PR 생성**: Github에 브랜치를 Push하고, `main` 브랜치로 병합해 달라는 PR을 엽니다.  
    PR 생성 시 자동으로 나타나는 템플릿의 체크리스트를 꼼꼼히 채워주세요!  
4. **코드 리뷰**: 최소 1명 이상의 스터디원에게 리뷰(Approve)를 받아야 합니다.  
5. **Merge**: 리뷰가 완료되고 승인(Approve)을 받으면, 코드를 `main`에 병합합니다.  

### 📝 커밋 메시지 컨벤션
#### ✅ 1. 커밋 유형 지정
**커밋 유형은 아래 규칙에 맞춰 작성해 주세요.**  
| Commit Type | 설명 |
| --- | --- |
| ✅ ::  | 프로젝트 세팅 |
| ⚡️ :: | 기능 업데이트 |
| ⚰️ :: | 기능 삭제 |
| 🐛 :: | 버그 수정 |
| 🔀 :: | Merge or PR |
| ♻️ :: | 코드 리펙토링 |
| 📄 ::  | 파일 생성 |
| 💉 :: | 테스트 코드 작성 |
| 🚀 :: | 새 버전 릴리즈 |
| 🎨 :: | 코드 스타일 |  

예시: `✅ :: [Seohyun] 무무 프로젝트 AI 모델 초기 세팅`
